## Overview

This project contains solutions to various LeetCode problems categorized by data structures and algorithms. Each
solution is implemented in Java with comprehensive documentation and follows best practices for code clarity and
maintainability.

**Project Origin & Purpose:** This repository started as a personal documentation effort to track the LeetCode questions
I studied. It served as my own reference guide for algorithm and data structure concepts. Now, others may fork it for
their own use to expedite learning or recapping algorithms and data structures.

## Topics Covered

- **Array**: Manipulation, searching, sorting, and other array operations
- **Binary Tree**: Traversal, construction, validation, and various tree algorithms
- **Sliding Window**: Fixed and variable size window techniques
- **Two Pointers**: Various two-pointer technique applications
- **Backtracking**: Recursive solutions for combinatorial problems
- **Binary Search**: Efficient searching in sorted arrays
- **Binary Search Trees**: BST operations and algorithms
- **Dynamic Programming**: Memoization and tabulation techniques
- **Graph Algorithms**: Traversal, shortest path, and connectivity problems
- **Greedy Algorithms**: Optimization problems solvable by greedy approaches
- **Hash Maps/Sets**: Efficient lookups and frequency counting
- **Heaps**: Priority queue applications
- **Intervals**: Merging, intersection, and other interval operations
- **Linked Lists**: Singly/doubly linked list manipulations
- **Math**: Mathematical computations and number theory problems
- **Matrix**: 2D array operations and transformations
- **Stack**: LIFO data structure applications

## How to Use

1. Clone the repository:
   ```
   git clone <repository-url>
   ```

2. Navigate to the specific problem category you're interested in:
   ```
   cd src/com/leetcode/practice/solution/[category]
   ```

3. Each solution class contains:
    - Problem name in the comment header
    - LeetCode problem URL
    - Detailed function documentation
    - Clean, readable implementation

## Coding Style

- All functions include detailed Javadoc-style comments
- Problem names are clearly marked in square brackets
- Time and space complexity are noted where applicable
- Consistent variable naming conventions
- Clear, readable code with appropriate comments
